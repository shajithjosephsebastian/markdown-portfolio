Linux Command: mv

Complete Beginner to Advanced Guide

üìå Overview

The mv (Move) command is used to move or rename files and directories in Linux. It is a core command for file management and organization.

Category: File Management
Difficulty: Beginner

üßæ Syntax

mv [OPTIONS] SOURCE DESTINATION

The source is the file or folder you want to move, and the destination is the new location or new name.

üü¢ Basic mv Commands (Beginner)

1. Rename a File

mv oldname.txt newname.txt

Renames a file from oldname.txt to newname.txt.

2. Move File to Another Directory

mv file.txt /home/user/Documents/

Moves the file to the specified directory.

3. Move Multiple Files

mv file1.txt file2.txt /home/user/Documents/

Moves multiple files into a destination folder.

4. Move a Directory

mv folder/ /home/user/Backup/

Moves an entire directory to another location.

üü° Intermediate Options

Command Description
mv -i file.txt dest/ Prompt before overwriting files
mv -v file.txt dest/ Show verbose output of the move operation
mv -n file.txt dest/ Do not overwrite an existing file
mv -u file.txt dest/ Move only if the source is newer than destination

üîµ Advanced mv Options (Pro Level)

Command Use Case
mv --backup file.txt dest/ Create a backup of destination file before overwriting
mv -t DEST SOURCE Specify target directory explicitly
mv -f file.txt dest/ Force move without prompting

üî• Real-World Examples

Rename a Folder

mv Projects Old_Projects

Move Files to Backup Directory

mv *.log /home/user/backup/

Move and Show Output

mv -v report.txt /home/user/Documents/

‚ö†Ô∏è Overwriting Behavior

By default, mv will overwrite files in the destination without warning. Use -i to get a confirmation prompt before overwriting important files.

üß† Pro Tips for Beginners

  • Use mv to both move and rename files
  • Use -i to prevent accidental overwrites
  • Use -v to see what files are being moved
  • Wildcards (*, ?) can be used to move multiple files at once

‚ùó Common Mistakes

  • Accidentally overwriting important files
  • Wrong destination path
  • Forgetting quotes for filenames with spaces
  • Example: mv "My File.txt" /home/user/
